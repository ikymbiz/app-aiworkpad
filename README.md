# app-aiworkpad

AIWorkPadは、ブラウザ上で動作するAIアシスタント内蔵のMarkdown対応スマートテキストエディタです。
文章を選択するだけで直感的にAIメニューを呼び出し、文章のブラッシュアップや要約、構造化、さらには深掘り調査までをシームレスに行うことができます。作成したデータはすべてブラウザのローカルストレージ（IndexedDB）に安全に保存され、オフラインライクで軽快な動作を実現しています。

モバイル端末とPCの両方に最適化されたレスポンシブデザインを採用しており、環境を問わず快適に執筆作業に集中できます。

---

## 💡 主な機能一覧

### 🤖 AIアシスタント機能（テキスト選択で瞬時に起動）
エディタ上のテキストを選択すると、AIアクションシートが自動でポップアップします。
- **きれい化**: 誤字脱字を修正し、冗長な表現を省いて文章を整えます。
- **要約**: 選択した長文を簡潔に要約し、重要なポイントを抽出します。
- **表現修整**: 文章の表現をより専門的、かつ魅力的にブラッシュアップします。
- **構造化**: ベタ打ちのテキストを、見出しや箇条書きを用いた適切なMarkdown形式に自動整形します。
- **深掘り調査**: 選択したキーワードや文章を元にAIが調査を実行し、詳細なレポートを生成します。

### 📝 リッチなドキュメント・執筆サポート
- **複数ドキュメント管理**: 左サイドバーから簡単に新しいドキュメントを作成・切り替え・削除可能。
- **自動目次生成 (TOC)**: 文章中にMarkdownの見出し（`#`）を入力すると、右サイドパネルにリアルタイムで目次が生成され、クリックで該当箇所へジャンプできます。
- **調査ノート**: 「深掘り調査」の結果は別タブの「調査ノート」としてストックされ、ワンクリックで本文中の好きな場所に挿入できます。

### 💾 安心のデータ保存とバージョン管理
- **完全ローカル保存**: `localForage` を利用し、ドキュメントデータや調査ノートをブラウザ内部に自動保存。
- **オートセーブ**: 入力が1秒止まると自動で変更内容を保存します。
- **タイムトラベル（履歴管理）**: 5分ごとの定期保存やAI処理実行前、手動保存時に自動でバージョン履歴を作成。いつでも過去のバージョンに復元可能です。

### 📱 クロスデバイス・レスポンシブUI
- PCでは広々とした3ペイン（サイドバー・エディタ・パネル）構成。
- スマートフォンでは、スワイプやボタン操作で引き出せるボトムシート・オーバーレイメニューを採用。

## 🛠 使用技術・ライブラリ

本プロジェクトはビルドツールを必要とせず、単一のHTMLファイルとして動作するよう設計されています。

- **HTML5 / CSS3 / Vanilla JavaScript**: ピュアなフロントエンド技術。
- **Tailwind CSS (CDN)**: 高速で柔軟なUIスタイリング。
- **FontAwesome (CDN)**: 直感的なUIアイコン。
- **localForage**: IndexedDBをラップし、大容量かつ非同期なローカルデータ保存を実現。
- **Marked.js**: AIが生成したMarkdown形式の調査ノートをHTMLにパースしてプレビュー表示。
- **Cloudflare Workers**: AI機能のバックエンドAPIとして通信（外部エンドポイントへPOSTリクエストを送信）。

## 🚀 セットアップ・使い方

### 1. 実行方法
特別なインストールやビルド環境は不要です。
1. リポジトリのファイル（`index.html`など）をダウンロードします。
2. ダウンロードしたHTMLファイルをGoogle Chrome、Safari、Edgeなどのモダンブラウザで直接開くだけでアプリが起動します。
   *※より安定した動作確認のためには、VS Codeの「Live Server」などのローカルサーバー環境での起動を推奨します。*

### 2. 基本的な使い方
1. **文章を書く**: 画面中央のエディタエリアにテキストを入力します。Markdown記法が使用できます。
2. **AIを利用する**: エディタ内のテキストをドラッグ（または長押し）して選択します。画面下部（PCでは選択付近）にAIアクションシートが表示されるので、目的のボタンをクリックします。
3. **調査ノートの活用**: 深掘り調査を実行すると、右サイドパネルの「調査」タブに結果が保存されます。結果を確認し、「本文に挿入」ボタンでエディタに取り込むことができます。
4. **過去の履歴に戻る**: 右サイドパネルの「履歴」タブを開き、戻りたい時点の「このバージョンを復元」をクリックします。

### 3. APIエンドポイント（Worker）の変更について
ソースコード内の `WORKER_URL` には、AIのリクエストを処理するための外部APIエンドポイントが指定されています。
ご自身のAI環境（OpenAI APIなど）を利用する場合は、JavaScriptの初期設定部分にある以下の定数を、ご自身で構築したAPIエンドポイントのURLに変更してください。

```javascript
// 定数・初期設定
const WORKER_URL = 'https://your-custom-worker.workers.dev';
```

API側は、`{"prompt": "送信されるプロンプト文字列"}` というJSONを受け取り、JSON形式（`{"response": "..."}`等）またはプレーンテキストで回答を返す仕様を満たす必要があります。
